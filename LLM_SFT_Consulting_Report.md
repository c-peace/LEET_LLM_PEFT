# 9B 모델 객관식 선지 생성 SFT 학습 및 데이터 증강 전략 컨설팅

## 목차
1. 현재 상황 진단 및 목표 설정
2. 1단계: 1,000개 데이터를 활용한 초기 SFT 학습 전략
3. 2단계: LLM을 활용한 데이터 증강(Data Augmentation) 전략
4. 3단계: 파인튜닝 최적화 및 하드웨어 효율화
5. 평가 및 모니터링 체계 (Evaluation)
6. 컨설턴트 종합 의견 및 Next Step

## 1. 현재 상황 진단 및 목표 설정
* **현재 상황:** 9B 파라미터 규모의 최신 오픈소스 모델(예: Llama 3 8B, Gemma 2 9B, Qwen 등)을 타겟으로, 특정 지문과 문제, 문제 유형을 바탕으로 5지선다 객관식 선지(정답 1, 오답 4)를 생성하는 SFT를 준비 중. 확보된 데이터 약 1,000 세트.
* **학습 목표:** 지문 내용의 논리와 문제의 의도에 부합하는 정답과, 지나치게 쉬운 오답이 아닌 수험생을 헷갈리게 할 수 있는 "매력적인 오답(Plausible Distractor)"을 생성하는 모델 개발.
* **전문가 진단:** 현재 보유하신 1,000개의 고품질 데이터는 모델이 **'5지선다 생성'이라는 포맷팅(Formatting)과 Task의 기본 형식을 이해(Instruction Following)하는 데 매우 적합하고 충분한 양**입니다. 
하지만 다양한 텍스트 도메인(법률, 언어, 과학 등)과 복잡한 문제 유형의 추론 능력을 모델에 완전히 내재화하기 위한 '일반화(Generalization) 성능'을 달성하기에는 데이터 양이 부족할 수 있습니다. 
따라서 제안하신 **"초기 소규모 SFT \-\> 모델 분석 \-\> 상위 LLM을 이용한 데이터 증강 \-\> 대규모 재학습"** 방식은 최신 LLM 파이프라인에서 가장 강력하고 효과적인 방법론입니다.

## 2. 1단계: 1,000개 데이터를 활용한 초기 SFT 학습 전략
초기 학습(Base SFT)의 최우선 목적은 모델에게 "당신은 문제를 출제하는 위원이고, 출력은 반드시 이런 형식이어야 한다"는 규칙을 각인시키는 것입니다.

* **프롬프트 템플릿(Chat Template) 최적화:** 
  현재 데이터셋(instruction, input, output)을 타겟 9B 모델에 맞는 Chat 포맷(예: ChatML 등)으로 매핑합니다. 
  * **System Prompt:** *"당신은 시험 출제 위원입니다. 주어진 지문과 문제 유형을 바탕으로 가장 적절한 정답 1개와 매력적인 오답 4개 로 구성된 객관식 선지 5개를 서술어 형태로 생성하세요."* 와 같이 명확한 역할과 제약을 부여해야 합니다.
* **하이퍼파라미터(Hyperparameters) 추천:**
  * **Epoch:** 1,000개 데이터의 경우 3~4 Epoch 정도로 시작하여 Overfitting을 방지합니다. Loss 그래프와 Validation 성능을 모니터링하며 조절합니다.
  * **Learning Rate (LR):** LoRA 사용 시 1e-4 ~ 2e-5 수준으로 설정하고 코사인 스케줄러(Cosine Scheduler)를 적용합니다.
* **1차 평가 목표:** 이 단계에서는 오답의 매력도보다 "항상 서술어로 끝나는가?", "정답과 오답이 겹치지 않는가?", "JSON 혹은 요구된 배열(Array) 형식의 출력을 깨짐 없이 반환하는가?"라는 **구조적 온전성** 달성에 집중합니다.

## 3. 2단계: LLM을 활용한 데이터 증강(Augmentation) 전략
1차 모델 학습 결과를 토대로 한계점을 분석한 후, GPT-4o, Claude 3.5 Sonnet 등 최고 성능의 상용 LLM을 활용해 데이터를 증강(Synthetic Data Generation)합니다.

* **Seed Data 기반 Prompt 다변화:**
  주어진 1,000개의 데이터를 Seed로 사용하여, 상위 모델에게 "이 지문을 바탕으로 '추론형' 문제를 새로 만들어 파생 선지를 생성해달라", "원문 지문을 다른 도메인으로 변형해서 선지를 구성해달라" 등의 방식으로 다양성을 확보합니다.
* **매력적인 오답(Plausible Distractor) 생성 파이프라인:**
  단순히 무관한 내용을 오답으로 만들면 모델의 품질이 낮아집니다.
  1. 지문에 등장한 단어들의 위치나 주체/객체를 교묘하게 바꾼 선지 (역관계, 인과관계 왜곡)
  2. 일반적 상식으로는 맞지만 '지문에는 없는' 내용
  3. 지문 내용의 범위를 과도하게 일반화하거나 한정하는 선지
  위와 같은 세밀한 규칙을 상위 LLM 프롬프트에 제공하여 고수준의 오답 데이터를 만듭니다.
* **Auto-Filtering & Verification (자기 정제 과정 최소화):**
  증강된 수만 개의 데이터 중 "정답이 2개가 되는 오류"나 "정답이 없는 경우"를 걸러야 합니다. 또 다른 비평가(Critic) LLM 파이프라인을 구축하여 생성된 문제-선지 세트에 스스로 답변하게 해보고 검증하여야 고품질의 SFT 데이터를 유지할 수 있습니다.

## 4. 3단계: 파인튜닝 최적화 및 하드웨어 효율화
9B 모델의 SFT를 단일 GPU 또는 제한된 환경에서 효과적으로 수행하기 위한 기술적 제안입니다.

* **PEFT (LoRA / QLoRA) 활용:** 
  Full Fine-Tuning 대신 4-bit / 8-bit 양자화를 결합한 QLoRA를 적극 권장합니다. Rank(r) 값은 16~64, Alpha 값은 32~128 사이로 잡아 타겟 모듈(q_proj, v_proj, o_proj 등)에 폭넓게 적용하세요.
* **시퀀스 패킹(Sequence Packing):** 
  지문의 길이가 다양할 텐데, ConstantLengthDataset 등을 이용하여 시퀀스 공간의 낭비를 최소화하면 GPU 효율이 크게 상승합니다.
* **NEFTune (Noise Embedding) 적용:** 
  SFT 성능 향상 기법 중 하나인 엠베딩 층에 미세 노이즈 추가(예: alpha=5)를 고려하면 모델이 프롬프트 포맷에만 과적합되는 것을 막고 조금 더 유연하고 다양한 표현을 생성할 수 있습니다.

## 5. 평가 및 모니터링 체계 (Evaluation)
텍스트 생성, 특히 '선지 생성'의 평가는 손실 함수(Loss)만으로는 품질을 담보할 수 없으므로 다각적 평가가 필요합니다.

* **정량적 형식 평가 (Rule-based Rule):**
  * 선지가 정확히 5개 생성되었는가?
  * 모든 선지가 '~다', '~음/기' 형태의 명확한 서술어로 마무리되었는가?
* **LLM-as-a-Judge (정성 평가 자동화):**
  * GPT-4o를 평가자로 활용해 (1) 지문과의 내용 부합성, (2) 오답의 함정 수준(매력도), (3) 문맥적 자연스러움을 1~5점 척도로 스코어링하는 체계를 갖춥니다.
* **Human Baseline Evaluation:**
  * 최종적으로 연구자 혹은 출제자가 직접 증강된 모델의 결과를 블라인드 테스트하여 실제 수험/모의고사에 활용할 수 있을 수준인지 정성적으로 평가해야 합니다.

## 6. 컨설턴트 종합 의견 및 Next Step
현재 기획하신 **"Seed 데이터(1K)로 포맷을 학습하고 \-\> LLM으로 지식 및 추론 다양성을 증강해 \-\> 대용량 2차 SFT를 수행하는 전략"**은 현재 업계에서 도메인 특화 SLM(Small Large Model)을 구축할 때 사용하는 가이드라인에 완벽히 부합합니다.

* **Action Item 1:** 보유하신 1,000개 데이터를 `Train 900` / `Test 100` 비율로 분리하십시오.
* **Action Item 2:** 900개의 데이터로 9B 모델에 초기 LoRA 학습을 돌려 Baseline 모델을 확보하십시오.
* **Action Item 3:** 확보한 Baseline 모델로 100개의 Test 데이터를 생성해보고, 모델이 어떤 오답을 잘 만들지 못하는지(ex. 너무 티가 나는 오답 등) 오류 유형을 정리하여 데이터를 증강할 프롬프트를 설계하십시오.
* **결론:** 훌륭한 초기 아이디어를 가지고 계십니다. 1단계 작업을 빠르게 프로토타이핑(Prototyping) 하시고 LLM 기반 검증 파이프라인(Verification)에 가장 많은 신경을 쏟으신다면, 우수한 객관식 선지 생성 자동화 AI를 확보하실 수 있을 것입니다.

## 7. 연구 목적 강화를 위한 5가지 심화 제안 (Research Contributions)
단순한 SFT를 넘어 상위 거대 모델(GPT-4 등)의 API 활용과 비교해 **독자적인 학술적/연구적 가치**를 확보하기 위한 5가지 제안입니다.

1. **목적 특화 지식 증류 (Task-Specific Knowledge Distillation):** 거대 상용 모델의 능력을 9B 소형 모델로 전이할 때의 파라미터 대비 성능 향상폭(효율성) 및 특정 도메인(LEET, PSAT)에서의 비용 우위 증명.
2. **난이도 제어 및 매력적인 오답 생성 (Controllable Distractor Generation):** 상위 모델이 단순하고 정직한 오답을 만드는 점을 지적하고, 조건부 학습(Conditional Training)을 통해 오답의 매력도와 난이도를 모델이 스스로 조절하게 하는 기법 연구.
3. **해설 동시 생성 및 자기 검증 (Self-Verification via Rationale Generation):** 선지뿐 아니라 "왜 오답/정답인지"에 대한 해설(Chain-of-Thought)을 동시에 학습시켜, 선지의 논리적 환각(Hallucination) 오류율 감소 효과 분석.
4. **약점 기반 능동 학습 추구 (Active Learning in Data Augmentation):** 맹목적 데이터 증강이 아닌, 1차 학습된 9B 모델이 취약한 논리 구조 및 문제 유형만 식별하여 해당 데이터만 상위 LLM으로 증강하는 데이터 효율성(Data Efficiency) 최적화 연구.
5. **범용 모델의 도메인 맹점 극복 분석 (Overcoming Domain Blind-spots):** 일반 거대 모델이 고도의 논리 문항 도메인(LEET 등)에서 범하는 어휘적/추론적 오류를 분석하고, SFT된 9B 모델이 이를 어떻게 학술적으로 교정하는지 정성/정량적 Error Analysis 수행.
